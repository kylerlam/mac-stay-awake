# Mac Stay Awake — Product Requirements Document

## 1. 文档信息

- 产品名称：Mac Stay Awake
- 产品形态：原生 macOS 菜单栏与窗口应用
- 当前版本：1.1.0
- 文档状态：发布基线
- 最后更新：2026-07-17

## 2. 产品概述

Mac Stay Awake 让用户无需记忆命令行，即可在“正常模式”和“合盖运行模式”之间切换。它的核心场景是在 MacBook 合盖后，继续运行 Codex、下载、编译或其他长时间后台任务。

当前经过用户实际验证的后端是：

- 开启：`pmset -a disablesleep 1`
- 关闭：`pmset -a disablesleep 0`
- 检测：读取 `pmset -g` 中的 `SleepDisabled`

IOPM 闲置休眠 assertion 与上述设置不等价，不能替代本产品的合盖运行后端。

## 3. 产品目标

- 一次点击开启或关闭合盖运行。
- 以系统实际值而不是内存布尔值决定 UI 状态。
- 只有确认 `SleepDisabled = 1` 后才显示绿色成功状态。
- 当用户请求的状态与实际值不一致时，给出清晰的橙色提示和恢复操作。
- 不向普通用户暴露 `SleepDisabled` 等命令行术语。
- 保持窗口、菜单栏图标和命令行检测结果一致。

## 4. 非目标与边界

- 不代替关机、重启或系统因电量耗尽、热保护等原因采取的保护动作。
- 不保证 Codex 或网络服务永不掉线；进程、网络和服务本身仍可能故障。
- 不使用 IOPM idle-sleep assertion 或 `caffeinate -i` 代替 `pmset disablesleep`。
- 当前版本不包含定时关闭、排程、登录时启动、通知或历史记录。

## 5. 功能需求

### FR-01 读取真实状态

应用必须通过 `/usr/bin/pmset -g` 读取 `SleepDisabled`，并在以下时机检测：

- 应用启动时；
- 控制窗口打开时；
- 系统唤醒后；
- 每次开启或关闭命令执行后；
- 用户点击“重新检测”时；
- 控制窗口可见期间每 5 秒一次。

### FR-02 开启合盖运行

- 用户点击“开启合盖运行”后，应用通过管理员授权执行 `pmset -a disablesleep 1`。
- 命令结束后必须重新读取系统值。
- 只有实际值为 `1` 时，界面才进入绿色“合盖运行模式”。
- 命令失败或读回值仍为 `0` 时，界面显示橙色不一致状态，并允许重试或接受实际的正常模式。

### FR-03 恢复正常模式

- 用户点击“恢复正常模式”后，应用通过管理员授权执行 `pmset -a disablesleep 0`。
- 命令结束后必须重新读取系统值。
- 只有实际值为 `0` 时，界面才进入灰色“正常模式”。
- 命令失败或读回值仍为 `1` 时，界面显示橙色不一致状态，并允许重试或保持当前合盖运行状态。

### FR-04 检测失败

- 无法运行或解析 `pmset -g` 时，界面显示橙色“无法确认系统状态”。
- 用户可点击“重新检测”。
- 检测失败时不得显示绿色成功状态。

### FR-05 退出清理

- 如果本次应用运行期间曾成功开启合盖运行，正常退出时应尝试恢复 `SleepDisabled = 0`。
- 如果应用只是检测到外部已经开启的状态，不应在退出时擅自修改该外部状态。

## 6. 状态与交互规格

| 实际/预期状态 | UI 状态 | 颜色 | 主要操作 | 次要操作 |
| --- | --- | --- | --- | --- |
| `SleepDisabled = 0` | 正常模式 | 灰色 | 开启合盖运行 | 重新检测 |
| `SleepDisabled = 1` | 合盖运行模式 | 绿色 | 恢复正常模式 | 重新检测 |
| 预期开启、实际为 `0` | 合盖运行未生效 | 橙色 | 重新开启合盖运行 | 使用正常模式 |
| 预期关闭、实际为 `1` | 未能恢复正常模式 | 橙色 | 重新恢复正常模式 | 保持合盖运行 |
| 无法读取实际值 | 无法确认系统状态 | 橙色 | 重新检测 | 无 |

颜色不是唯一的状态信号；所有状态必须同时使用文字和图标表达。

## 7. 技术要求

- 语言：Swift 6
- UI：SwiftUI 与必要的 AppKit 窗口集成
- 平台：macOS 14 或更高版本
- 开启/关闭命令：`/usr/bin/osascript` 请求管理员权限后运行 `/usr/bin/pmset`
- 状态来源：`/usr/bin/pmset -g`
- 状态模型、视图与系统服务保持职责分离。
- UI 操作进行期间屏蔽重复操作。
- 应用不收集、存储或传输个人数据。

## 8. 验收标准

### AC-01 启动同步

启动应用后，UI 必须读取并显示当前系统实际值；不能仅因内存默认值而显示绿色或灰色。

### AC-02 开启读回

执行开启命令后，只有读回 `SleepDisabled = 1` 才显示绿色“合盖运行模式”。

### AC-03 关闭读回

执行关闭命令后，只有读回 `SleepDisabled = 0` 才显示灰色“正常模式”。

### AC-04 不一致恢复

当请求值和实际值不一致时，UI 必须显示橙色，并同时提供重试和接受实际状态的操作。

### AC-05 自动与手动检测

窗口可见时每 5 秒自动检测；用户可随时点击“重新检测”；系统唤醒后自动检测。

### AC-06 命令行一致性

应用显示的“防止系统休眠”状态必须与 `script/status.sh` 所查看的 `SleepDisabled` 值一致。

### AC-07 合盖场景

开启后应使用当前已经验证的 `pmset disablesleep` 行为支持合盖运行。发布说明不得把普通 IOPM assertion 描述成等价替代方案。24 小时连续在线仍需在真实电源和网络条件下由用户观察验证。

## 9. 测试与发布验证

- 单元测试覆盖正常、开启、关闭、不一致、检测失败和退出清理。
- `swift test` 必须通过。
- `./script/build_and_run.sh --verify` 必须成功构建应用包并启动进程。
- 发布前验证应用包版本号、最低系统版本、签名状态和 DMG 内容。
- DMG 必须同时包含 `MacStayAwake.app` 和指向 `/Applications` 的快捷方式。
